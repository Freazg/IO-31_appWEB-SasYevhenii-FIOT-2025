1. ОПИС ПРЕДМЕТНОГО СЕРЕДОВИЩА

Загальний опис

Трекер звичок (Habit Tracker) - це веб-застосунок для формування та відстеження корисних звичок користувачів. Застосунок допомагає людям систематично виконувати необхідні дії, відстежувати прогрес і підтримувати мотивацію через візуалізацію досягнень.

Чому це важливо?

Дослідження показують, що для формування нової звички потрібно в середньому 21-66 днів регулярного виконання. Наш застосунок допомагає пройти цей шлях, надаючи інструменти для відстеження, мотивації та аналізу прогресу.

Основні сутності системи

  • Користувач - особа, яка використовує систему для відстеження звичок
  • Звичка - регулярна дія, яку користувач хоче виробити
  • Запис виконання - фіксація факту виконання звички в конкретний день
  • Статистика - узагальнені дані про виконання звичок за період
  • Категорії - групування звичок

Функціонал системи

Управління звичками:

  • Створення нової звички з параметрами
  • Редагування існуючих звичок
  • Видалення звичок
  • Архівування завершених звичок

Відстеження виконання:

  • Позначення звички як виконаної за день
  • Скасування позначки виконання
  • Додавання нотаток до виконання
  • Перегляд історії виконання

Візуалізація та аналітика:

  • Календар виконання - кольорова карта активності
  • Статистика серій - кількість днів поспіль
  • Загальний прогрес у відсотках
  • Графіки та діаграми досягнень

2. ОПИС БІЗНЕС-ЛОГІКИ

Основні сценарії роботи користувача:

Сценарій 1: Реєстрація та налаштування профілю

  1. Користувач заходить на сайт
  2. Проходить реєстрацію
  3. Налаштовує профіль
  4. Вибирає час нагадувань

Сценарій 2: Створення першої звички

  1. Натискає кнопку "Додати звичку"
  2. Заповнює форму
  3. Встановлює параметри
  4. Зберігає звичку

Сценарій 3: Щоденне відмічання виконання

  1. Користувач заходить у додаток
  2. Бачить список активних звичок
  3. Натискає галочку біля виконаної звички
  4. Система оновлює статистику

3. ФУНКЦІОНАЛЬНІ ВИМОГИ

FR-1: Управління звичками

  • FR-1.1: Система повинна дозволяти створювати нову звичку
  • FR-1.2: Система повинна дозволяти редагувати звичку
  • FR-1.3: Система повинна дозволяти видаляти звичку

FR-2: Відстеження виконання

  • FR-2.1: Система повинна дозволяти відмічати виконання звички
  • FR-2.2: Система повинна зберігати історію виконання
  • FR-2.3: Система повинна розраховувати серії виконання

FR-3: Візуалізація даних

  • FR-3.1: Система повинна відображати календар виконання
  • FR-3.2: Система повинна показувати графіки прогресу
